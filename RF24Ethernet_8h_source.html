325  * @page RF24EthernetOverview RF24Ethernet Overview
326  * @section Overview Overview
327  *
328  * The RF24Ethernet library was initially designed as an experiment and potential testing tool for [RF24Network](http://nRF24.github.io/RF24Network), an OSI Layer 3 network driver, allowing a Raspberry Pi to
329  * act as a TCP/IP gateway or host for connected sensor nodes. An Arduino can interface with any Linux machine or SLIP capable device supporting USB, or
330  * preferably, an RPi runs companion software, [RF24Gateway](http://nRF24.github.io/RF24Gateway/), which creates a network interface linked to the RF24 radio network. This interface can be
331  * further linked to the local network or internet. This allows the RPi or Arduino-based gateway to perform automatic discovery and routing of TCP/IP data,
332  * with no required pre-configuration or interaction from the user beyond assigning appropriate addresses to the nodes initially.
333  *
334  * @section What What does it do?
335  *
336  * RF24Ethernet creates a network of internet enabled RF24/Arduino sensors. It provides an API and interface very similar to the Arduino Ethernet library,
337  * to allow sensor nodes to connect out to local or internet based devices to retrieve or send information, or be connected to from the internet or your
338  * local network directly.
339  *
340  * Sensor nodes can act as individual web servers, simple command-and-control servers, or can connect out as required via TCP/IP.
341  *
342  *
343  * @section How How does it work?
344  *
345  * RF24Ethernet utilizes the UIP TCP/IP stack, allowing Arduino devices to use a Raspberry Pi running [RF24Gateway](http://nRF24.github.io/RF24Gateway/) or Arduino
346  * as a gateway to your network or the internet, or simply as a repository for sensor information. The RF24, RF24Network and optionally RF24Mesh libraries
347  * handle the underlying routing, addressing etc. so users do not need to be familiar with the radio modules or libraries.
348  *
349  * RF24Network addresses need to be assigned as MAC addresses, and IP addresses can be configured as desired by the user. The master node (00) uses
350  * either the Address Resolution Protocol (ARP) or RF24Mesh to find the appropriate nodes when IP traffic is sent though, and routes traffic to the correct
351  * RF24Network address.
352  *
353  * This provides a fairly seamless interaction, since users only need to configure standard IP forwarding and firewall rules as desired.
354  *
355  * The RF24 libraries are based on the [OSI model](http://en.wikipedia.org/wiki/OSI_model):<br>
356  * | System | OSI Layer | Description |
357  * |------------------------|------------------------|---------------------------------------------------------------------------|
358  * | <b>NRF24L01 Radio Modules</b> | OSI Layer 1 (Physical) | Transmission and reception of the raw bit stream over the physical medium. |
359  * | <b>RF24 Core Radio Driver</b> | OSI Layer 2 (Data Link)| Transfer of data frames over the physical link. Establish/Terminate logical links between nodes. Manage acknowledgements, error checking |
360  * | <b>RF24Network</b> | OSI Layer 3 (Network) | Routing, subnets, subnet traffic control, logical-to-physical address mapping, frame fragmentation/reassembly.|
361  * | <b>RF24Ethernet (uIP)</b> | OSI Layer 4 (Transport)| Ensures error-free messages. Manages sequencing, losses, retries, and duplicates. |
362  * | <b>RF24Ethernet (core)</b> | OSI Layer 5 (Session) | Establish, maintain, and terminate connections |
363  * | <b>RF24Mesh</b> -Optional- | OSI Layer 7 (Application)| Provides DHCP/DNS type protocols and allows dynamic addressing/topology for RF24Network.
364  *
365  *
366  *
367  * **TAP vs TUN:**
368  *
369  * RF24Ethernet and RF24Gateway are able to utilize both TAP and TUN type interfaces. <br>
370  * <b> TAP -</b> A TAP interface can be looked at much the same as an Ethernet interface, Ethernet headers are used to identify devices via MAC address, and the Address
371  * Resolution Protocol (ARP) is used to perform that identification. <br>
372  * <b> TUN -</b> A TUN interface does not utilize Ethernet headers, MAC addresses or ARP, in this case relying on RF24Mesh/IP routing instead.
373  *
374  * **Limitations:**
375  *
376  * UDP frames can be up to 512 bytes in length, so UDP/DNS is limited by available memory, and the defined MAX_PAYLOAD_SIZE / UIP_BUFFER_SIZE. If a
377  * DNS request exceeds the maximum allowed size, it will be dropped. DNS responses for addresses such as 'www.google.com' tend to be much smaller than requests
378  * for 'google.com', and will generally work with the default configuration.
379  *
380  * **MAC address formatting:**
381  *
382  * RF24Ethernet uses a simple method of formatting the MAC addresses, using the first four bytes to store the characters 'RF24', and the last two bytes to store the
383  * RF24Network address, which makes identification of nodes via MAC address very simple using standard monitoring tools.
384  *
385  * | RF24Network Address| Byte 0 | Byte 1 | Byte 2 | Byte 3 | Byte 4 | Byte 5 |
386  * |--------------------|--------|--------|--------|--------|--------|--------|
387  * | n/a | R | F | 2 | 4 |Octal LB|Octal HB|
388  * | 01 | 0x52 | 0x46 | 0x32 | 0x34 | 0x01 | 0x00 |
389  * | 011 | 0x52 | 0x46 | 0x32 | 0x34 | 0x09 | 0x00 |
390  * | 0443 | 0x52 | 0x46 | 0x32 | 0x34 | 0x23 | 0x01 |
391  *

401  * <b> RPi </b>
402  *
403  * On the Raspberry Pi, a companion program, <a href="http://nRF24.github.io/RF24Gateway/">RF24Gateway</a> must be installed along with the RF24 and RF24Network libraries
404  * 1. @code wget http://tmrh20.github.io/RF24Installer/RPi/install.sh @endcode
405  * 2. @code chmod +x install.sh @endcode
406  * 3. @code ./install.sh @endcode
407  * 4. @code cd RF24Gateway/examples/ncurses @endcode
408  * 5. @code make @endcode
409  * 6. @code sudo ./RF24Gateway_ncurses @endcode
410  * 7. The application will require the user to specify an IP address and Subnet Mask: 10.10.2.2 and 255.255.255.0 are the defaults with RF24Ethernet examples
411  * 8. Raspberry Pi defaults to the master node (00) using RF24Mesh. Secondary Raspberry pi nodes need to specify their RF24Network address or RF24Mesh nodeID.
412  *
413  *<b> Arduino </b>
414  * 1. For Arduino devices, use the Arduino Library Manager to install the RF24 libraries
415  * 2. Open the included Getting_Started_SimpleServer or Getting_Started_SimpleClient example
416  * 3. Configure your chosen CE and CS pins for the radio connection.
417  * 4. Configure the RF24Mesh nodeID (Any unique value from 3 to 255)
418  * 5. Configure the IP address according to your preferences, (last octet must==nodeID) with the gateway set to the chosen IP of the RPi.
419  * 6. Connect into your nodes web-server at http://ip-of-your-node:1000 from the RPi or configure the client sketch to connect to a server
420  * running on the Raspberry Pi.
421  * Note: To minimize memory usage on Arduino, edit RF24Network_config.h with a text editor, and uncomment `#define DISABLE_USER_PAYLOADS`. This
422  * will disable standard RF24Network messages, and only allow external data, such as TCP/IP information. Remember to comment for normal operation!
423  *
424  * <b> Non-Raspberry Pi (Linux etc) Devices </b><br>
425  * Arduino can also function as a gateway for any Linux machine or PC/MAC that supports SLIP. <br>
426  * See the SLIP_Gateway and SLIP_InteractiveServer
427  * examples for usage without the need for a Raspberry Pi.
428  *
429  * <b>Accessing External Systems: Forwarding and Routing</b>
430  *
431  * In order to give your network or nodes access to your network or the internet beyond the RPi, it needs to be configured to route traffic
432  * between the networks.
433  * 1. Run @code sudo sysctl -w net.ipv4.ip_forward=1 @endcode to allow the RPi to forward requests between the network interfaces
434  * 2. @code sudo iptables -t nat -A POSTROUTING -j MASQUERADE @endcode to allow the RPi to perform NAT between the network interfaces <br>
435  *
436  * @note This configuration is generally for initial testing only. Users may also need to add a static route to their local machine, or configure port forwarding on the RPi.<br>
437  * See the following links for more information on configuring and using IPTables:<br>
438  * http://www.karlrupp.net/en/computer/nat_tutorial<br>
439  * http://serverfault.com/questions/326493/basic-iptables-nat-port-forwarding
440  *
441  * @warning <b>Note:</b> Users are responsible to manage further routing rules along with their IP traffic in order to prevent unauthorized access.

449  * @page CustomNetworks Building an RF24Ethernet Network
450  *
451  *
452  * The general configuration instructions set up a basic sensor network, with static nodes, using TCP/IP over RF24Network. The layered
453  * library design also allows some nodes to communicate solely at the RF24Network level, creating hybrid networks to extend range over
454  * large distances, or improve connectivity in distant areas.

457  * @section RF24Mesh RF24Mesh Networks
458  *
459  * RF24Mesh allows dynamic configuration of RF24Network addresses, and allows the creation of dynamic sensor networks. Nodes are able
460  * to move around physically, releasing and renewing their RF24Network address as required. Once configured with a unique nodeID, nodes using RF24Mesh
461  * can automatically configure themselves and join the network as required, with no user interaction.
462  *
463  * See the included <a href = SimpleServer_mesh_8ino-example.html>SimpleServer_Mesh</a> example for general usage and configuration information
464  * with RF24Ethernet.
465  *
466  * **RF24Mesh** - Additional Information
467  *
468  * When utilizing RF24Mesh along with RF24Ethernet, relay/routing nodes should be configured with RF24Mesh, or designated on the master node as static nodes
469  * with a pre-configured RF24Network address.
470  *
471  * RF24Mesh can be used with TAP/Ethernet configurations, but utilizing TUN/RF24Mesh together will reduce overhead.
472  *
473  * See the links below for more information on utilizing RF24Mesh with RF24Ethernet and RF24Network.
474  *
475  * @section RF24Gateway RF24Gateway
476  *
477  * See http://nRF24.github.io/RF24Gateway/ for more information on RF24Gateway
478  *
479  ** @section HybridNetworks Hybrid- RF24Network/RF24Ethernet Networks
480  *
481  * The default configuration of RF24Ethernet and RF24Network enables both TCP/IP and the underlying messaging protocols provided by RF24Network.
482  *
483  * Relay/Routing nodes can be configured using only the RF24 and RF24Network layers (optionally RF24Mesh), and will automatically handle data
484  * accordingly, whether it is an external data type (TCP/IP) or an internal network message from another RF24Network node. This allows very small and
485  * inexpensive devices such as ATTiny to act as relay/routing nodes when extending network range and/or connectivity.
486  *
487  * Arduino/AVR devices can utilize custom or included sleep modes in RF24Network, and benefit from the lower power consumption and low-power features
488  * of the nrf24l01 chips directly.
489  *
